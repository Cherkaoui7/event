🎯 1. Présentation du projet

DOMINATORES est la première plateforme digitale marocaine dédiée à l'organisation d'événements sur mesure. Développée dans le cadre du module PIE, elle couvre l'intégralité du parcours de planification — de la simulation à la confirmation de réservation.

🏛️

Personnalisation de salle

Décoration (zellige, traditionnel, moderne), éclairage, disposition des tables — tout configurable en temps réel.

🎮

Simulateur 3D interactif

Visualisation Three.js avec React Three Fiber. Caméra orbite, changement de décoration en direct, export photo.

🍲

Traiteur marocain

Catalogue complet : couscous, tajines, pastilla, méchoui, pâtisseries. Prix calculé automatiquement par invité.

💡

Suggestions intelligentes

L'IA recommande la meilleure configuration selon le budget, le type d'événement et le nombre d'invités.

📦

Packs tout inclus

Packs préconfigurés (Économique, Classique, Luxe) — appliquent salle + traiteur en un seul clic.

🪑

Plan de table interactif

Disposition visuelle des tables selon le nombre d'invités et la configuration choisie.

📄

Devis PDF

Génération d'un récapitulatif complet imprimable / exportable en PDF directement depuis le navigateur.

💳

Paiement simulé

Modale de paiement d'acompte (30%) qui confirme la réservation et change le statut de l'événement.

🇲🇦
Projet conçu pour le marché marocain avec des contenus, tarifs et styles adaptés à la culture locale (zellige, motifs marocains, plats traditionnels, monnaie MAD).

⚙️ 2. Architecture technique

Le projet adopte une architecture découplée : le backend Laravel expose une API REST, et le frontend React consomme cette API de manière indépendante.

Navigateur
React 19 + Vite
localhost:5173
⟷
API REST JSON
Laravel 10
localhost:8000
⟷
Stockage
SQLite
database.sqlite

Stack Backend

Laravel 10 PHP 8.1+ SQLite Laravel Sanctum (Auth Bearer) Eloquent ORM API Resources

Stack Frontend

React 19 Vite 8 React Router 7 Three.js + React Three Fiber @react-three/drei Recharts Axios CSS-in-JS (inline styles)

Communication & Sécurité

  • Authentification : Token Bearer via Laravel Sanctum (stocké dans localStorage)
  • Proxy Vite : les appels /api/* du frontend sont redirigés vers http://localhost:8000
  • Rôles : client (accès à ses événements) et admin (accès au panneau d'administration)
  • Validation : toutes les entrées sont validées côté serveur (Laravel Form Validation)

Étape 1 — Cloner le projet

bash 
git clone https://github.com/votre-repo/dominatores.git
cd dominatores

Étape 2 — Backend Laravel

bash — dossier /backend 
cd backend

# Installer les dépendances PHP
composer install

# Copier le fichier d'environnement
cp .env.example .env          # Linux / Mac
copy .env.example .env        # Windows

# Générer la clé d'application
php artisan key:generate

Étape 3 — Configurer SQLite

bash — créer le fichier SQLite 
# Linux / Mac
touch database/database.sqlite

# Windows PowerShell
New-Item -Path database\database.sqlite -ItemType File

Ouvrez .env et configurez :

.env 
DB_CONNECTION=sqlite
DB_DATABASE=/chemin/absolu/vers/backend/database/database.sqlite

# Windows exemple :
# DB_DATABASE=C:\\Users\\Vous\\dominatores\\backend\\database\\database.sqlite

SANCTUM_STATEFUL_DOMAINS=localhost:5173
SESSION_DOMAIN=localhost
⚠️
Important : Le chemin DB_DATABASE doit être un chemin absolu. Sur Windows, utilisez des double-backslashes \\ ou des slashs normaux /.

Étape 4 — Migrations & Données initiales

bash 
# Créer toutes les tables
php artisan migrate

# Insérer les données : plats marocains + packs événementiels
php artisan db:seed

Étape 5 — Créer le compte Administrateur

bash — Laravel Tinker 
php artisan tinker

// Coller cette commande dans Tinker :
App\Models\User::create([
    'name'     => 'Admin',
    'email'    => 'admin@dominatores.ma',
    'password' => bcrypt('password123'),
    'role'     => 'admin',
]);

exit

Étape 6 — Frontend React

bash — dossier /frontend 
cd ../frontend

# Installer toutes les dépendances
npm install

# Vérifier que les packages 3D sont présents :
npm install three @react-three/fiber @react-three/drei recharts
📦
Packages clés : three + @react-three/fiber + @react-three/drei pour la 3D, et recharts pour les graphiques du dashboard admin.

🚀 5. Lancer l'application

Deux terminaux doivent rester ouverts en parallèle :

Terminal 1 — Serveur API (Laravel)

bash — /backend 
cd backend
php artisan serve
✅
L'API démarre sur http://localhost:8000 — toutes les routes /api/* sont accessibles.

Terminal 2 — Application React (Vite)

bash — /frontend 
cd frontend
npm run dev
✅
L'application React démarre sur http://localhost:5173 — ouvrez cette adresse dans votre navigateur.
🌐
Pour la 3D interactive, utilisez un navigateur moderne avec WebGL activé : Chrome, Firefox ou Edge. Safari peut présenter des limitations.

✨ 6. Fonctionnalités du site

6.1 — Page d'accueil

Présentation de la plateforme avec une mise en page immersive (motif zellige en fond). Les visiteurs non connectés voient les boutons Simulation gratuite et Créer un compte. Les clients connectés accèdent directement à leur tableau de bord.

6.2 — Inscription & Connexion

Système d'authentification sécurisé par token. Le rôle client est attribué automatiquement à l'inscription. Le rôle admin est assigné manuellement via Tinker.

6.3 — Tableau de bord client

Vue centralisée de tous les événements du client avec statuts (Brouillon / Confirmé / Annulé), prix total, et accès rapide aux actions :

  • Créer un nouvel événement
  • Accéder à la personnalisation (4 onglets)
  • Consulter le récapitulatif / devis PDF

6.4 — Création d'événement

Formulaire simple et guidé : titre de l'événement, type (mariage, anniversaire, fête, professionnel), nombre d'invités, budget maximum. À la validation, l'utilisateur est redirigé vers la personnalisation.

6.5 — Personnalisation de la salle (4 onglets)

🏛️ Salle
🍲 Traiteur
📦 Packs
🪑 Plan de table
Onglet Contenu
🏛️ Salle Sélection du style de décoration (Zellige, Marocain classique, Moderne épuré), de l'éclairage (tamisé, spots, lustres traditionnels) et de la disposition des tables. Un aperçu 3D interactif se met à jour en temps réel en bas de page, avec option d'export photo.
🍲 Traiteur Catalogue organisé par catégories (entrées, plats, desserts). Ajout de plats avec sélection de la quantité (défaut = nombre d'invités). Le prix total se recalcule automatiquement après chaque modification.
📦 Packs Packs préconfigurés (Mariage Économique, Mariage Luxe, Anniversaire Festif). Cliquer sur « Appliquer » configure automatiquement la salle ET le traiteur, remplace les choix précédents et recalcule le prix.
🪑 Plan de table Visualisation interactive du placement des tables selon le nombre d'invités et la disposition choisie. Permet de simuler l'organisation spatiale de l'événement.

La barre de prix en bas de page se met à jour en temps réel après chaque modification. L'enregistrement est automatique.

6.6 — Simulateur 3D

Accessible via l'onglet Salle ou la navigation principale. Construit avec Three.js / React Three Fiber :

  • Vue orbite : cliquez et faites glisser pour tourner autour de la salle
  • Zoom avec la molette de la souris
  • Changement de décoration, éclairage et disposition en temps réel
  • Bouton "Prendre une photo" pour capturer la vue 3D
⚠️
La 3D nécessite un navigateur avec WebGL activé. Sur Chrome, accédez à chrome://settings/system et vérifiez que "Utiliser l'accélération matérielle" est activé.

6.7 — Suggestions intelligentes

Formulaire de simulation accessible sans créer d'événement. L'utilisateur entre :

  • Type d'événement
  • Budget maximum disponible
  • Nombre d'invités prévu

Le système analyse ces paramètres et recommande automatiquement une configuration optimale. Un bouton "Créer cet événement" pré-remplit l'événement avec la configuration suggérée.

6.8 — Récapitulatif & Devis PDF

Page de synthèse complète reprenant toutes les options sélectionnées avec les prix unitaires et le total. Deux actions sont disponibles :

  • 📄 Exporter en PDF : utilise la fonction d'impression du navigateur pour générer un PDF professionnel
  • 💳 Payer l'acompte (30%) : modale de paiement simulé qui confirme la réservation et passe le statut à « Confirmé »

6.9 — Dashboard Administration

Accessible uniquement aux comptes admin via /admin. Contient trois onglets :

Onglet Contenu
📊 Tableau de bord KPIs (total utilisateurs, total événements, revenu total, revenu moyen). Graphiques : donut (répartition types d'événements, statuts), courbe du revenu mensuel, histogramme des inscriptions, top 5 clients par revenus.
👥 Utilisateurs Liste paginée de tous les utilisateurs avec nom, email, rôle et date d'inscription.
📅 Événements Liste de tous les événements avec client associé, type, statut et prix total.

🔌 7. API REST — Endpoints

Routes publiques Pas d'auth

Méthode URL Description
POST /api/register Inscription (retourne un token)
POST /api/login Connexion (retourne un token)
POST /api/suggestions Obtenir une suggestion selon budget

Routes authentifiées Bearer Token

Méthode URL Description
POST /api/logout Déconnexion (révoque le token)
GET /api/user Profil de l'utilisateur connecté
GET /api/events Liste des événements du client
POST /api/events Créer un événement (statut: draft)
GET /api/events/{id} Détails + salle + traiteur
PUT /api/events/{id} Modifier / confirmer un événement
GET /api/events/{id}/room Configuration actuelle de la salle
POST /api/events/{id}/room Sauvegarder la configuration salle
GET /api/catering Catalogue des plats marocains
POST /api/events/{id}/catering Ajouter un plat à l'événement
PUT /api/events/{id}/catering/{itemId} Modifier la quantité d'un plat
DEL /api/events/{id}/catering/{itemId} Retirer un plat
GET /api/packs Liste des packs disponibles
POST /api/events/{id}/apply-pack Appliquer un pack à l'événement
GET /api/decorations Catalogue des décorations

Routes Admin Admin uniquement

Méthode URL Description
GET /api/admin/users Liste complète des utilisateurs
GET /api/admin/events Liste complète des événements
GET /api/admin/stats Statistiques pour le dashboard (KPIs, graphiques)

🗄️ 8. Structure de la base de données

Table Colonnes principales Description
users id, name, email, password, role Utilisateurs — rôle client ou admin
events id, user_id, title, event_type, status, guest_count, budget, total_price Événements — statuts : draft / confirmed / cancelled
rooms id, event_id, table_layout, decoration_style, lighting_style, custom_options Configuration de la salle (1 salle par événement)
catering_items id, name, description, category, price_per_person, active Catalogue des plats marocains
event_catering id, event_id, catering_item_id, quantity, line_total Plats sélectionnés par événement (table pivot)
packs id, name, event_type, base_price, decoration_style, lighting_style, table_layout Packs préconfigurés
pack_catering id, pack_id, catering_item_id, quantity_per_guest Plats inclus dans chaque pack
decorations id, name, type, price, active Catalogue des éléments de décoration
room_decorations id, room_id, decoration_id, quantity, line_total Décorations choisies par salle
personal_access_tokens tokenable_id, token, abilities, expires_at Tokens Sanctum (générés automatiquement)

Relations clés

  • Un utilisateur possède plusieurs événements
  • Un événement a exactement une salle (relation 1:1)
  • Un événement peut avoir plusieurs plats via event_catering
  • Un pack regroupe plusieurs plats via pack_catering
  • Une salle peut avoir des décorations supplémentaires via room_decorations

❓ 10. FAQ & Dépannage

Le frontend ne charge pas les données de l'API

Vérifiez :

  • Que le serveur Laravel tourne : php artisan serve doit afficher "Application running on http://localhost:8000"
  • Que vite.config.js contient bien le proxy /api → http://localhost:8000
  • Que vous ouvrez http://localhost:5173 et non un autre port

Erreur "no such table: events" (ou autre table)

bash — réinitialiser complètement 
# Recrée toutes les tables ET recharge les données :
php artisan migrate:fresh --seed

# Puis recréer l'admin :
php artisan tinker
App\Models\User::create(['name'=>'Admin','email'=>'admin@dominatores.ma','password'=>bcrypt('password123'),'role'=>'admin']);
exit

Erreur 401 Unauthorized sur tous les appels API

Le token a expiré ou est invalide. Solutions :

  • Déconnectez-vous et reconnectez-vous depuis l'interface
  • Ou videz le localStorage dans les outils développeur (F12 → Application → Local Storage)
  • Vérifiez que SANCTUM_STATEFUL_DOMAINS=localhost:5173 est dans le .env

La page /admin affiche "Accès non autorisé"

Vérifiez que le middleware est enregistré dans app/Http/Kernel.php :

app/Http/Kernel.php 
protected $routeMiddleware = [
    // ...
    'admin' => \App\Http\Middleware\AdminMiddleware::class,
];

Le simulateur 3D ne s'affiche pas

  • Vérifiez l'installation : npm install three @react-three/fiber @react-three/drei
  • Utilisez Chrome ou Firefox (Safari a des problèmes WebGL)
  • Dans Chrome, activez l'accélération matérielle : chrome://settings/system
  • Vérifiez la console navigateur (F12) pour les erreurs Three.js

Les graphiques du dashboard admin ne s'affichent pas

bash 
npm install recharts

Puis vérifiez que la route GET /api/admin/stats retourne des données depuis le backend.

Erreur CORS au démarrage

Ouvrez config/cors.php et vérifiez :

config/cors.php 
'paths'            => ['api/*', 'sanctum/csrf-cookie'],
'allowed_methods'  => ['*'],
'allowed_origins'  => ['http://localhost:5173'],
'allowed_headers'  => ['*'],
'supports_credentials' => false,

Réinitialisation complète rapide

bash 
# === Backend ===
cd backend
php artisan migrate:fresh --seed
php artisan tinker
App\Models\User::create(['name'=>'Admin','email'=>'admin@dominatores.ma','password'=>bcrypt('password123'),'role'=>'admin']);
exit
php artisan serve

# === Frontend (autre terminal) ===
cd frontend
npm run dev
🎉
Après ces commandes, l'application est prête. Ouvrez http://localhost:5173 et connectez-vous avec admin@dominatores.ma / password123.
